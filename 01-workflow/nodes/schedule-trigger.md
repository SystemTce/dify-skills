# 定时触发器节点详解

## 概述

定时触发器使工作流能够按指定时间自动运行，非常适合定期任务，如每日报告生成或定时通知。

## 核心功能

- 按固定时间表自动执行工作流
- 支持简单和复杂的时间调度
- 使用 Cron 表达式实现灵活配置
- 更新系统时间戳变量

## 关键约束

- 仅适用于工作流应用
- 每个工作流最多一个定时触发器
- 不产生输出变量，但会更新 `sys.timestamp` 系统变量

## 配置方式

### 可视化配置

适用于简单的时间模式：

```yaml
- data:
    type: schedule-trigger
    schedule:
      type: visual
      pattern: daily
      time: "09:00"
  id: daily_trigger
```

**支持的模式**:
- 每小时 (hourly)
- 每天 (daily)
- 每周 (weekly)
- 每月 (monthly)

### Cron 表达式配置

适用于复杂的时间需求：

```yaml
- data:
    type: schedule-trigger
    schedule:
      type: cron
      cron: "0 9 * * 1-5"  # 工作日上午9点
  id: cron_trigger
```

## Cron 表达式详解

### 基本结构

```
* * * * *
│ │ │ │ │
│ │ │ │ └─ 星期几 (0-6, 0=周日)
│ │ │ └─── 月份 (1-12)
│ │ └───── 日期 (1-31)
│ └─────── 小时 (0-23)
└───────── 分钟 (0-59)
```

### 特殊字符

**星号 (*)**: 匹配所有值
```
* * * * *  # 每分钟执行
```

**逗号 (,)**: 指定多个值
```
0 9,12,18 * * *  # 每天 9:00, 12:00, 18:00
```

**连字符 (-)**: 指定范围
```
0 9-17 * * *  # 每天 9:00 到 17:00 每小时
```

**斜杠 (/)**: 指定步长
```
*/15 * * * *  # 每 15 分钟
0 */2 * * *   # 每 2 小时
```

**L**: 最后一个
```
0 0 L * *     # 每月最后一天
0 0 * * 5L    # 每月最后一个周五
```

**问号 (?)**: 不指定值
```
0 0 15 * ?    # 每月 15 日（不关心星期几）
```

### 预定义快捷方式

```yaml
@yearly   # 等同于 0 0 1 1 *   (每年 1 月 1 日)
@monthly  # 等同于 0 0 1 * *   (每月 1 日)
@weekly   # 等同于 0 0 * * 0   (每周日)
@daily    # 等同于 0 0 * * *   (每天午夜)
@hourly   # 等同于 0 * * * *   (每小时)
```

## 常见 Cron 示例

```yaml
# 每天上午 8:30
0 8 30 * * *

# 工作日上午 9:00
0 9 * * 1-5

# 每周一上午 10:00
0 10 * * 1

# 每月 1 日和 15 日上午 9:00
0 9 1,15 * *

# 每 30 分钟
*/30 * * * *

# 每天上午 9:00 到下午 5:00，每小时
0 9-17 * * *

# 每季度第一天（1月、4月、7月、10月）
0 0 1 1,4,7,10 *

# 每月最后一天
0 0 L * *
```

## 系统变量

### sys.timestamp

每次触发器执行时更新：

```yaml

    type: template
    template: "执行时间: {{#sys.timestamp#}}"
  id: log_time
```

## 常见问题

### Q: 如何选择合适的执行频率？

A:
- 考虑数据更新频率
- 评估任务执行时间
- 平衡及时性和资源消耗
- 从较低频率开始，根据需要调整

### Q: Cron 表达式不生效怎么办？

A:
1. 使用在线工具验证表达式
2. 检查字段顺序和值范围
3. 确认特殊字符使用正确
4. 查看执行日志了解错误信息

### Q: 如何处理时区问题？

A:
- 确认服务器时区设置
- 在 Cron 表达式中考虑时区差异
- 使用 UTC 时间避免夏令时问题
- 在文档中明确标注时区

### Q: 定时任务失败了怎么办？

A:
1. 检查工作流日志
2. 验证触发器配置
3. 确认下游节点正常
4. 实现错误通知机制
5. 设置备用执行方案

### Q: 可以暂停定时触发器吗？

A:
- 在已发布的工作流中可以禁用触发器
- 使用快速设置启用/禁用
- 不需要重新发布工作流
- 禁用后不会执行，但配置保留

## 性能考虑

### 执行频率
- 每分钟执行：适合实时监控
- 每小时执行：适合定期检查
- 每天执行：适合日常报告
- 每周/月执行：适合周期性任务

### 资源消耗
- 高频执行会增加系统负载
- 长时间运行的任务需要更多资源
- 并发执行可能导致资源竞争
- 考虑使用队列机制

### 优化建议
- 批量处理减少执行次数
- 使用缓存避免重复计算
- 异步处理耗时操作
- 监控资源使用情况
